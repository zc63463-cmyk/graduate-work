# Audit Reports Directory

Use this directory for project-wide audit and review reports. Patch implementation records belong in `reports/patches/`; experiment-specific reports belong in `reports/experiments/`.

Historical root-level Markdown reports have been moved under `reports/` and are indexed in `reports/REPORT_INDEX.md`.

Suggested naming:

- `YYYYMMDD_TOPIC_AUDIT.md`
- `YYYYMMDD_TOPIC_REVIEW_NOTES.md`

Do not move runtime files, experiment CSVs, generated figures, or thesis assets into this directory.
