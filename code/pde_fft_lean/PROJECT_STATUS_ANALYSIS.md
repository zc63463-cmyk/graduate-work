### 2.1 🔴 A1 Bug：非齐次 Dirichlet BC 修正符号错误

**已确认存在**。搜索结果显示代码中共 **9 处** `F -= bc/h²` 需改为 `F += bc/h²`：

| `helmholtz_solver.py` | 319 | `F[0, :] -= bc_l_full / h**2` | `+=` |
| `helmholtz_solver.py` | 320 | `F[-1, :] -= bc_r_full / h**2` | `+=` |
| `gmres_solver.py` | 391 | `F[0, :] -= bc_l[1:-1] / h2` | `+=` |

| `gmres_solver.py` | 393-394 | `F[:, 0] -= bc_b[1:-1] / h2` + `F[:, -1] -= bc_t[1:-1] / h2` | `+=` |
| `gmres_solver.py` | 418-419 | `F[0, :] -= bc_l / h2` + `F[-1, :] -= bc_r / h2` | `+=` |

**当前未被检测到的原因**：所有测试用齐次 Dirichlet BC（g=0），`F -= 0` 和 `F += 0` 等价。

**数学推导验证**：

五点格式在点 (1,j)（邻左边界）：
```
(u_{0,j} - 4u_{1,j} + u_{2,j}) / h² + (u_{1,j-1} - 4u_{1,j} + u_{1,j+1}) / h² + k²u_{1,j} = f_{1,j}
```
其中 u_{0,j} = g_{0,j}（边界值）。移项到右端：
```
(-4u_{1,j} + u_{2,j} + u_{1,j-1} + u_{1,j+1}) / h² + k²u_{1,j} = f_{1,j} - g_{0,j}/h²
```

等等——这取决于矩阵的符号约定。让我再仔细检查：

代码中矩阵 A 代表的是 (-Δ + k²) 离散化，即 A = -Lh_5pt + k²I。五点 Laplacian 离散化 Lh_5pt 的 (1,j) 行：
- 对角：4/h²（中心点）
- off-diagonal：-1/h²（四个邻居）
- 边界邻居 u_{0,j} = g_{0,j} 被移到右端

方程 A·u = f 变为：
```
(4/h² + k²)u_{1,j} - u_{2,j}/h² - ... = f_{1,j} + g_{0,j}/h²
```
所以右端修正确实是 **加号**：`F += bc/h²`。

但等等，这取决于 A 的定义。如果 A = Lh_5pt（正定，Δ 的离散化）：
- 对角：-4/h²
- off-diagonal：1/h²

那修正就是减号。需要看代码中矩阵是怎么构建的。

实际上，审稿报告的推导更仔细。五点格式中，邻边界点方程包含边界值项。关键是：
- 离散方程左边含 `-g_{0,j}/h²`（作为 off-diagonal 乘边界值）
- 移项后：`RHS = f + g_{0,j}/h²`

**但** 代码使用 `F -= bc/h²`，与正确的 `F += bc/h²` 相反。**这是一个真实的 Bug。**

### 2.2 check_resonance 确认

代码中 `check_resonance` 计算 `ltot = LXX + LYY + k2`，对 modified Helmholtz（+k²）这个值永远 > 0，所以永远不会报警。这与审稿 A4 完全一致。

### 2.3 FACR 实现确认

代码 `facr_helmholtz` 的步骤是：
1. 1D transform in x → O(N² log N)
2. l 步 CR on y-line
3. solve reduced systems
4. back-substitute
5. inverse transform in x → O(N² log N)

这确实是"先 DST 后 CR"的实现，总复杂度至少 O(N² log N)，达不到声称的 O(N² log log N)。

## 四、核心诊断与 Lean 4 的角色

### 4.1 根本问题（审稿报告的精辟总结）

> **论文定义了 modified Helmholtz (-∇²+k²)u=f，但行文时按 true Helmholtz (-∇²-κ²)u=f 的困难来讨论。**

这在三个层面造成矛盾：

| 层面 | 现象 | Lean 4 验证的贡献 |
|------|------|-------------------|
| **理论** | 共振条件 λ+k²=0 永不成立 | `modified_no_resonance`：形式化证明 λ+k²>0 |
| **实验** | GMRES 迭代次数随 k² 增大而减少 | 未直接验证（实验数据问题） |
| **结论** | 声称"大波数困难"与数据矛盾 | `true_helmholtz_resonance_iff`：确认共振只来自 λ-k²=0 |

### 4.2 Lean 4 的验证价值

1. **A5 (Lh≈Δ)**：这是最容易出错的符号约定。Lean 形式化确认了 λ_L(0,0) = 0，排除 Lh≈-Δ 的可能性。
2. **A2 (double counting)**：`kron_eigen_point` 确认 2D 特征值 = μ+ν，不是 4+4。
3. **A4 (modified H. 无共振)**：这是论文最根本的问题。Lean 给出了最简洁的形式化证明。
4. **A10 (六阶证明)**：旧 Lean 项目已验证 c₂ 可消去（α=-γ, β=γ），与审稿指出的问题完全一致。

### 4.3 Lean 4 的局限

1. **实空间 BC 修正**（A1）：Lean 只校验了频域，不涉及实空间离散化。
2. **算法复杂度**（A6）：复杂度分析不是代数恒等式，无法用 Lean 验证。
3. **实验数据一致性**（A3）：需要人工对照，不是形式化验证的范畴。
4. **DCT-I 正交性**（A9）：涉及矩阵正交性的完整形式化，远超当前项目范围。
