# Lean 4 形式化验证：六阶紧致差分不可行定理

## 概述

使用 Lean 4 (v4.20.0) 对论文定理 2.3（六阶紧致差分格式不可行性）进行了形式化验证。
所有定理均通过 Lean 4 内核验证，零错误零警告。

## 形式化文件

- **路径**: `code/lean4_formalization/SixthOrderImpossibility.lean`
- **依赖**: 仅 Lean 4 核心（Int + omega），无需 Mathlib
- **编译**: `lake env lean SixthOrderImpossibility.lean`

## 已验证的定理列表

### 核心定理

1. **`sixth_order_impossible`**: 不存在整数 γ 使得 c₄(ξ,η,γ) = 0 对所有模式成立
   - 证明：c₄(1,0,γ) = 0 要求 60γ+3 = 0，c₄(1,1,γ) = 0 要求 120γ-4 = 0
   - 2·(60γ+3) - (120γ-4) = 10 ≠ 0，矛盾

2. **`poisson_no_sixth_order`**: 完整 Poisson 不可行性
   - c₂=0 要求 α=-γ，然后 c₄=0 不可能

3. **`c4_is_the_obstruction`**: c₄ 是六阶不可行的决定性障碍

### 辅助定理

4. **`c4_num_mode_10`**: N(1,0,γ) = 60γ + 3
5. **`c4_num_mode_11`**: N(1,1,γ) = 120γ - 4
6. **`c4_num_mode_01`**: N(0,1,γ) = 60γ + 3
7. **`c4_num_mode_21`**: N(2,1,γ) = 1020γ + 95
8. **`no_integer_gamma`**: 60γ+3=0 和 120γ-4=0 不兼容
9. **`no_rational_gamma_core`**: 有理数情形下的不可行性
10. **`no_rational_gamma`**: γ=a/d (d>0) 情形不可行
11. **`c2_at_10/01/11`**: c₂ 在特定模式下的值
12. **`c2_poisson_forces_alpha_neg_gamma`**: c₂=0 推出 α=-γ

## 数学要点

### c₄ 系数定义

修正紧致格式 `(L_h + αh²L₅)u - k²(R_h + βh²I)u = -(R_h + γh²I)f` 的 h⁴ 截断误差系数分子：

$$N(\xi, \eta, \gamma) = 3\eta^6 + 60\gamma\eta^4 - 5\eta^4\xi^2 - 5\eta^2\xi^4 + 60\gamma\xi^4 + 3\xi^6$$

### 关键洞察

1. c₄ 系数**不依赖于 k²**（当 c₂=0 参数 α=-γ, β=γ 被采用时）
2. 因此 Poisson 和 Helmholtz 方程的不可行性证明完全相同
3. 仅需两个模式 (1,0) 和 (1,1) 即可建立矛盾

### 证明的代数核心

```
c₄(1,0;γ) = 0  ⟹  60γ + 3 = 0   ⟹  γ = -1/20
c₄(1,1;γ) = 0  ⟹  120γ - 4 = 0  ⟹  γ = 1/30
-1/20 ≠ 1/30  ⟹  不存在通用 γ
```

等价地（避免分数运算）：2·(60γ+3) - (120γ-4) = 10 ≠ 0。

## 形式化方法说明

- 使用 `Int` 而非 `ℝ`（Lean 4 核心不包含实数类型）
- 使用 `omega` 策略处理线性整数算术
- 使用 `simp only [...]` 简化整数幂运算
- 有理数不可行性通过"消去分母"归约为整数情形
- Mathlib 版本（需要 `ring`、`linarith`）在文件末尾以注释形式记录

使用 `fft9_eigenvalue_proof.py` 进行符号验证：
- h² 系数 = 0 ✓（确认四阶精度）
- h⁴ 系数 = -(ξ²+η²)(3ξ⁴-8ξ²η²+3η⁴)/720 ✓
- 六阶不可行：c₂ 模式依赖（k²≠0）或 c₄ 模式依赖（k²=0）✓
